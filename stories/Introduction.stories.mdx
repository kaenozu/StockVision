# StockVision Component Library

StockVision Storybookへようこそ！このドキュメントには、StockVisionプロジェクトで使用されるすべてのUIコンポーネント、デザインシステム、および使用例が含まれています。

## 🎯 目的

このStorybookの目的：
- **コンポーネントの一元管理**: すべてのUIコンポーネントを一箇所で管理・閲覧
- **デザインシステムの統一**: 一貫したデザイン言語の維持
- **開発効率の向上**: 独立した環境でのコンポーネント開発・テスト
- **ドキュメント化**: コンポーネントの使用方法とプロパティの明確化
- **品質保証**: ビジュアル・機能テストによる品質管理

## 🏗️ アーキテクチャ

StockVisionプロジェクトは以下の構成になっています：

### フロントエンド（Web）
- **技術スタック**: React + TypeScript + Material-UI
- **状態管理**: Redux Toolkit
- **ルーティング**: React Router
- **スタイリング**: Emotion + Material-UI Theme

### モバイルアプリ
- **技術スタック**: React Native + TypeScript + React Native Paper
- **状態管理**: Redux Toolkit
- **ナビゲーション**: React Navigation
- **開発環境**: Expo

### 共通コンポーネント
両方のプラットフォームで共有される設計原則とコンポーネント構造

## 📱 プラットフォーム別対応

### Web Components
Material-UIベースのコンポーネント群：
- **ダッシュボード**: 株価データの総合ビュー
- **チャート**: インタラクティブな株価チャート
- **テーブル**: データ表示・フィルタリング
- **フォーム**: ユーザー入力・設定

### Mobile Components
React Native Paperベースのコンポーネント群：
- **ホームスクリーン**: モバイル最適化ダッシュボード
- **ナビゲーション**: タブ・スタックナビゲーション
- **カード**: 株価情報のカード表示
- **モーダル**: モバイル向けモーダル・アラート

## 🎨 デザインシステム

```css
/* プライマリー */
--primary: #2196F3 (Blue 500)
--primary-light: #90CAF9 (Blue 200)
--primary-dark: #1565C0 (Blue 800)

/* セカンダリー */
--secondary: #FF9800 (Orange 500)
--secondary-light: #FFB74D (Orange 300)
--secondary-dark: #F57C00 (Orange 800)

/* セマンティックカラー */
--success: #4CAF50 (Green 500)
--error: #F44336 (Red 500)
--warning: #FF9800 (Orange 500)
```

### タイポグラフィ
- **フォントファミリー**: Roboto (Web) / System Font (Mobile)
- **見出し**: 階層的なヘディングスケール (H1-H6)
- **本文**: 読みやすい行間・文字サイズ
- **コード**: Fira Code / Monaco (等幅フォント)

### スペーシング

- **md**: 16px
- **lg**: 24px
- **xl**: 32px
- **xxl**: 48px

## 🛠️ 使い方

### 1. コンポーネント閲覧
左側のサイドバーから興味のあるコンポーネントを選択してください。各コンポーネントには以下の情報が含まれています：
- **概要**: コンポーネントの目的と用途
- **プロパティ**: 利用可能なProps一覧
- **使用例**: 実際の使用パターン
- **バリエーション**: 異なる状態・スタイル

### 2. インタラクティブテスト
**Controls**パネルで：
- プロパティの値をリアルタイムで変更
- コンポーネントの挙動を確認
- 異なる状態のテスト

### 3. コード確認
**Story**タブで：
- 実際のReactコードを確認
- 実装方法の学習
- コードスニペットのコピー

### 4. デザイン確認
**Design System**パネルで：
- カラーパレット
- タイポグラフィスケール  
- スペーシングガイド
- コンポーネント使用ガイドライン

## 📊 コンポーネントステータス

各コンポーネントには以下のステータスが付いています：

- 🟢 **Stable**: 本番利用可能、APIが安定
- 🟡 **Beta**: 機能完成、APIが変更される可能性
- 🔴 **Alpha**: 開発中、大きな変更の可能性
- ⚫ **Deprecated**: 非推奨、新規利用は避ける

## 🔧 開発ガイドライン

### 新しいコンポーネントの追加
1. `/src/components/` にコンポーネント作成
2. `/stories/` にStorybookストーリー作成  
3. プロパティとバリエーションを定義
4. ドキュメント作成
5. テストの追加

### コンポーネント設計原則
- **単一責任**: 一つのコンポーネントは一つの責任
- **再利用性**: 複数の場所で使える設計
- **拡張性**: 将来の要件変更に対応
- **アクセシビリティ**: すべてのユーザーが利用可能
- **パフォーマンス**: 最適化された実装

### コードスタイル
- **TypeScript**: 型安全な実装
- **ESLint + Prettier**: 一貫したコードフォーマット
- **JSDoc**: 適切なコメント・ドキュメント
- **Testing**: ユニット・統合テストの実装
